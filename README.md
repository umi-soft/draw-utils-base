# 1. Web GIS 通用绘图工具

这是一个通用的图形绘制工具包，工具包内部独立维护图层的创建和销毁

# 2. 公共能力需求说明

工具抽象设计如下：

```typescript

// 数组元素顺序为[lng, lat]
type Coordinate = [number, number]

enum DrawState { IDLE, CREATING, EDITING }

abstract class IWebGisDrawBasicUtil {
    /**
     * @param map 地图实例由业务集成方实例化时传递
     * @param pickUpCallback 工具内部对于鼠标的单击、双击事件，若当前处于等待操作状态，且本次单击或双击命中了某个元素或多个元素，则回调该方法
     */
    constructor(map: any, pickUpCallback: (features: FeatureInfo[]) => void)

    /**
     * 单击事件，由外部业务集成方统一监听后，回调该方法
     * @param coord 经纬度坐标
     */
    abstract click(coord: Coordinate)

    /**
     * 双击事件，由外部业务集成方统一监听后，回调该方法
     * @param coord 经纬度坐标
     */
    abstract dblclick(coord: Coordinate)

    /**
     * 鼠标移动事件，由外部业务集成方统一监听后，回调该方法
     * @param coord 经纬度坐标
     */
    abstract move(coord: Coordinate)

    /**
     * 通过API直接添加一个要素，要检测id是否重复，若重复，直接返回false
     * @param info
     */
    abstract addFeature(info: FeatureInfo): boolean
    
    /**
     * 依据id，强行重置某个元素，需检测ID是否存在，若不存在，直接返回false
     * 若当前该元素正在被编辑，直接先关闭编辑，进入等待操作状态，然后整体重置这个元素
     * @param info
     */
    abstract modifyFeature(info: FeatureInfo): boolean

    /**
     * 1. 检测逻辑，检测当前必须处于等待状态，若不处于等待状态，直接返回false，业务集成方自行处理相关提示信息
     * 2. 相关初始化工作，AI思考需要执行哪些初始化逻辑
     * 3. 进入新增状态，返回true
     * @param info 符合第四章设计规范
     * @param callback 完成新增绘制时业务回调
     */
    abstract createFeature(info: FeatureInfo, callback?: (feature: FeatureInfo) => void): boolean

    /**
     * 1. 检测逻辑，检测当前必须处于等待状态，若不处于等待状态，直接返回false，业务集成方自行处理相关提示信息
     * 2. 其他准备工作, AI思考需要执行哪些初始化逻辑
     * 3. 进入编辑状态，返回true
     * @param info 符合第四章设计规范
     * @param callback 完成编辑绘制时业务回调
     */
    abstract updateFeature(info: FeatureInfo, callback?: (feature: FeatureInfo) => void): boolean

    /**
     * 删除指定要素
     * 1. 检测逻辑，检测当前必须处于等待状态，若不处于等待状态，直接返回false，业务集成方自行处理相关提示信息
     * 2. 执行删除逻辑, AI思考需要执行哪些初始化逻辑，恢复等待操作状态并返回true
     * @param id 需要被删除要素的ID信息
     * @param callback 删除成功后的业务回调
     */
    abstract deleteFeature(id: string | number, callback?: (feature: FeatureInfo) => void): boolean

    /**
     * 强制清空所有要素
     * 直接进入等待操作状态，清空所有要素后，返回true
     * @param callback 清空成功后的业务回调
     */
    abstract clear(callback?: (features: FeatureInfo[]) => void): boolean

    // 获取当前工具状态
    abstract getState(): DrawState

    /**
     * 销毁一切
     */
    abstract destroy()
}

```

## 2.1 图层需求

- 每次实例化时，工具负责图层的创建
- 工具支持并行实例化，允许同时存在多个实例（注意图层也是多实例的）
- 业务集成方销毁工具时，工具销毁图层、图层中的全部信息、交互过程中产生的所有信息
- 鼠标移动、单击、双击事件，由外部集成方统一回调当前工具对应方法

## 2.2 发起新建绘制 createFeature

1. 业务集成方每次发起新建绘制，每次绘制只允许绘制一个元素
2. 由业务集成方发起要素新建绘制，支持以下参数传递和约束

- 新增要素信息：FeatureInfo，符合第四章节规范，但：
- - wkt初始时是空的，完成绘制后有工具负责生成
- - name初始时可能也是空的，若为空则需要随机生成一段长度为8位的全局唯一的英文名称作为要素名称
- - 每种类型的图形，数据结构存在些许差异
- 绘制完成业务回调：callback，完成新增绘制后，回调告知业务集成方

3. 业务集成方 *有效* 发起要素新建绘制后

- 鼠标第一次双击事件为绘制的正式开始，进入新增状态
- 鼠标第二次双击为结束当前要素的绘制标记，恢复等待操作状态，触发新增业务回调函数，告知业务集成方绘制完成

4. 点类型要素特殊，第一次双击既是开始也是结束

## 2.3 发起编辑绘制 updateFeature

编辑绘制是面向利用工具对业务上历史产生的要素进行二次编辑场景

1. 有效进入编辑状态

- 复制一份要素信息，用于与实时编辑形成前后对比效果，原始展示展示，编辑完成后，保留最新即可
- 被编辑的要素，边和顶点均需要展示拖拽靶点，用于支撑拖拽编辑，拖拽行为，详见第三章各类型要素个性化渲染内容
- - 点类型要素除外，无需构建靶点，自身就是靶点
- 双击任意位置，视为编辑结束，恢复等待操作状态，回调 callback，告知业务集成方


## 2.4 面积和边长渲染要求

当前工具内所有要素，新增和编辑均支持实时可视化边长和面积
- 部分特殊类型的要素除外，可能不存在边或者面积概念，可视化渲染要求具体参见第三章每种图形的个性化渲染需求
- 边长计算：单位依据实际大小自适应（米、千米），精度2位小数，例如1.20m、1.01 km，计算具备实时效果，位于所在边的中心位置
- 面积计算：单位依据实际大小自适应（平方米、平方千米），精度2位小数，具备绘制实时效果，位于要素的中心点，向右偏移一点

## 2.5 名称和icon渲染

- icon可视化渲染，支持zoom动态长度宽度等比例变化，zoom越小，icon越小
- icon 位于要素中心点，向上偏移一点
- name可视化渲染，支持zoom动态大小调整，zoom越小，文字越小
- name 位于要素中心点，向下偏移一点

# 3. 各类型图形绘制

后续需要支持很多类图形的绘制能力，因此要合理的命名，增强代码可读性

## 3.1 点

要素类型 type: point

1. 绘制行为描述

- 双击即完成点的绘制
- 二次编辑行为，无需构建拖拽靶点，直接拖拽即可拖拽位置到新的位置

2. 个性化渲染

- 等待操作状态下，鼠标悬浮时，形成高亮效果，高亮时，放大1.5倍即可
- 无需考虑边长和面积问题，不存在这两个概念
- 对于 iconSrc 为空，视为无需icon，采用圆点模式渲染（默认）
- 对于 iconSrc 不为空，视为icon模式

3. 数据结构设计（JSON方式体现）：PointInfo

```json
{
    "name": "名称，允许为空，为空时，视为当前类型图形无需渲染 name",
    "iconSrc": "如果该字段为空，说明point不使用icon模式，使用圆点模式，会影响UI渲染逻辑，圆点模式是默认模式",
    "fillRgba": "填充颜色，RGBA格式",
    "textRgba": "文本颜色，RGBA格式，对name有效"
}
```

## 3.2 矩形

要素类型 type: rect

1. 绘制行为描述

- 新增矩形绘制过程中，鼠标第一次双击视为第一个顶点，此后鼠标位置代表对角线顶点，直至第二次双击确定对角线顶点，结束新增绘制
- 二次编辑则支持四个顶点均可以拖拽移动，拖拽过程中，以对角线顶点作为基准拖拽，形成新的矩形
- 二次编辑支持四个边拖拽，拖拽时，可理解为边在做上下或者左右平移
- 二次编辑始终保障是矩形

2. 个性化渲染

新增绘制和二次编辑，矩形变化时实时响应的

- 矩形的大小在编辑过程中是跟随鼠标移动变化的
- 在实时变化的工程中，动态计算面积和边长
- 边长展示涉及一个长边和一个短边即可
- 等待操作状态下，鼠标悬浮时，形成高亮效果，高亮时，矩形线条加粗 1.5 倍

3. 数据结构设计（JSON方式体现）：RectInfo

```json
{
    "name": "名称，允许为空，为空时，视为当前类型图形无需渲染 name",

    "lineWidth": "线条宽度",
    "lineType": "线条类型，虚线 dashed、实线 solid，虚线配置固定即可",
    "lineRgba": "线条颜色，RGBA格式",
    "fillRgba": "填充颜色，RGBA格式",
    "textRgba": "文本颜色，RGBA格式，对所有文字类生效"
}
```

## 3.3 三角形

要素类型 type: triangle

1. 绘制行为描述

- 鼠标第一次双击视为第一个顶点，下一次单击视为确定第二个顶点，确定第二个顶点后，鼠标位置代表第三个顶点，直至鼠标再次双击确定第三个顶点，结束新增绘制
- 二次编辑支持三个顶点均可以拖拽移动，拖拽过程中其余两个顶点位置不变
- 二次编辑支持三个边均可以拖拽，视为所选边的平移行为，此时该边的两个顶点同步移动
- 二次编辑始终保持是一个完整的三角形

## 3.6 折线

要素类型 type: polyline

1. 绘制行为描述

- 新增绘制过程中，鼠标第一次双击视为第一个点，并进入新增绘制状态，后续单击视为折线上的连续点，再次双击得到线段的最后一个点，结束新增绘制
- 约束当线段只有一个点时，再次双击视为无效双击，必须保障至少有3个点构成折线
- 二次编辑时，当前线段上任意点构成拖拽靶点，鼠标悬浮命中线段上任意位置，实时构建一个任意靶点，支持拖拽
- 二次编辑时，始终至多有一个任意靶点，鼠标移出则消失，悬浮在线段上方，则出现，单击后，开启这个任意靶点拖拽，松开即结束拖拽
- 任意靶点拖拽，不是形成边平移，而是在这个变中间追加了一个点

2. 个性化渲染

- 无需考虑面积，线段无面积概念
- 实时计算所有构成的连续线段的边长
- 等待操作状态下，鼠标悬浮时，形成高亮效果，线段线条放大1.5倍

# 4. 要素数据结构定义 FeatureInfo

- 留意detail，该字段是面向的要素编辑面板中的内容，具体要看每一类图形的详细定义
- 业务集成方和当前工具，传递要素空间信息，统一采用wkt标准

```typescript
interface FeatureInfo {
    "id": "要素ID",
    "type": "要素类型，以章节 3 支持的类型为准",
    "wkt?": "wkt格式的要素点字符串",
    "detail": ShapeInfo | PolylineInfo | LineInfo | CircleInfo | TriangleInfo | RectInfo | PointInfo
}
```
